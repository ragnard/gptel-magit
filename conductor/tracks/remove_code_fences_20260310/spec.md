# Track Spec: remove_code_fences_20260310

## Goal
The goal of this track is to refine the commit message generation process in `gptel-magit` by stripping away triple backtick Markdown code fences (e.g., ` ``` `) and any surrounding whitespace. Single backticks should be preserved.

## Problem Statement
Currently, `gptel-magit` takes the raw response from the LLM and formats it using `gptel-magit--format-commit-message`. If the LLM wraps the commit message in Markdown triple code fences (which it often does by default), those fences and any extra whitespace are included in the final commit message.

For example, an LLM response like:
```
 ``` 
 feat(core): add new feature
 ```
```
Should be transformed into:
```
feat(core): add new feature
```

## Proposed Changes
1.  **Refine `gptel-magit--format-commit-message`:** Modify this function (or add a pre-processing step) to:
    -   Identify and remove leading/trailing triple backtick code fences (` ``` `).
    -   Trim all leading and trailing whitespace from the resulting message.
    -   **Important:** Ensure single backticks used for code representation *within* the message are preserved.
2.  **Add Test Coverage:** Implement ERT tests to verify the removal of triple backtick code fences and whitespace for various LLM output patterns.

## Success Criteria
-   Commit messages generated by `gptel-magit` do not contain leading or trailing triple backtick code fences.
-   Leading and trailing whitespace is removed from the generated message.
-   Single backticks used within the message are preserved.
-   The core formatting (subject/body distinction) remains intact.
