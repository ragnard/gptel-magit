# Track Spec: remove_code_fences_20260310

## Goal
The goal of this track is to refine the commit message generation process in `gptel-magit` by stripping away common LLM formatting artifacts, specifically code fences (e.g., ` ``` `) and any surrounding whitespace.

## Problem Statement
Currently, `gptel-magit` takes the raw response from the LLM and formats it using `gptel-magit--format-commit-message`. If the LLM wraps the commit message in Markdown code fences (which it often does by default), those fences and any extra whitespace are included in the final commit message.

For example, an LLM response like:
```
 ``` 
 feat(core): add new feature
 ```
```
Should be transformed into:
```
feat(core): add new feature
```

## Proposed Changes
1.  **Refine `gptel-magit--format-commit-message`:** Modify this function (or add a pre-processing step) to:
    -   Identify and remove leading/trailing Markdown code fences (both ` ``` ` and single backticks).
    -   Trim all leading and trailing whitespace from the resulting message.
2.  **Add Test Coverage:** Implement ERT tests to verify the removal of code fences and whitespace for various LLM output patterns.

## Success Criteria
-   Commit messages generated by `gptel-magit` do not contain Markdown code fences.
-   Leading and trailing whitespace is removed from the generated message.
-   The core formatting (subject/body distinction) remains intact.
